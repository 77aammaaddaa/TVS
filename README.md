# Eco Fine Pro V14 - نظام ERP متكامل

<div dir="rtl">

## 📋 نظرة عامة

**Eco Fine Pro V14** هو نظام تخطيط موارد المؤسسات (ERP) متكامل الميزات، مصمم خصيصاً للشركات الصغيرة والمتوسطة. يوفر النظام حلاً شاملاً لإدارة جميع جوانب الأعمال من نقطة بيع، محاسبة، مخزون، علاقات عملاء، موارد بشرية، وأنظمة قانونية وتقسيط.

### ⚠️ تحذير أمني هام

> **هذا النظام غير جاهز للاستخدام في بيئة الإنتاج** بسبب ثغرات أمنية حرجة تم تحديدها أثناء المراجعة. يرجى قراءة قسم [الأمن](#-التقييم-الأمني) بعناية قبل أي نشر.

## ✨ المميزات الرئيسية

### 🎯 وحدات الأعمال

| الوحدة | الوصف |
|--------|-------|
| **نقطة البيع (POS)** | نظام بيع متكامل مع دعم العمل دون اتصال |
| **إدارة العلاقات (CRM)** | تتبع العملاء، المبيعات، والحملات التسويقية |
| **المحاسبة** | دفتر أستاذ، قيود يومية، تقارير مالية |
| **المخزون** | تتبع المنتجات، التنبيهات، الجرد |
| **الموارد البشرية** | إدارة الموظفين، الرواتب، الحضور |
| **النظام القانوني** | العقود، القضايا، المتابعة القانونية |
| **التقسيط** | إدارة خطط الدفع، المتابعة، التحصيل |
| **الخزينة** | إدارة التدفقات النقدية |
| **الموردين** | إدارة سلسلة التوريد |
| **التقارير** | لوحات معلومات وتحليلات شاملة |

### 🔧 المميزات التقنية

- ✅ **تصميم معياري**: هيكل قائم على الوحدات لسهولة الصيانة
- ✅ **دعم Multi-Tenant**: عزل بيانات العملاء
- ✅ **Offline-First**: يعمل بدون اتصال ثم يزامن
- ✅ **قاعدة بيانات محلية**: IndexedDB للتخزين المؤقت
- ✅ **توثيق عربي**: واجهة وتوثيق باللغة العربية
- ✅ **مزامنة تلقائية**: عند استعادة الاتصال

## 🚀 التثبيت والاستخدام

### المتطلبات الأساسية

- متصفح حديث يدعم ES6+ و IndexedDB
- اتصال إنترنت للتحقق من الترخيص والمزامنة
- حساب مستخدم صالح (يتم إنشاؤه عبر لوحة الإدارة)

### خطوات التشغيل

1. **استنساخ المستودع**
   ```bash
   git clone <repository-url>
   cd ecofine-pro-v14
   ```

2. **تشغيل خادم محلي** (اختياري)
   ```bash
   # باستخدام Python
   python -m http.server 8000
   
   # أو باستخدام Node.js
   npx serve
   ```

3. **الوصول للتطبيق**
   - افتح المتصفح وانتقل إلى `http://localhost:8000`
   - أو افتح ملف `index.html` مباشرة

4. **الإعداد الأولي**
   - إنشاء حساب مشرف فائق
   - تكوين إعدادات الشركة
   - تفعيل الرخصة

## 🔐 التقييم الأمني

### ⚠️ الثغرات الحرجة المكتشفة

| الثغرة | الخطورة | الوصف |
|--------|---------|-------|
| **كلمات مرور كنص صريح** | 🔴 حرجة | تخزين كلمات المرور بدون تشفير |
| **مفاتيح API مكشوفة** | 🔴 حرجة | مفاتيح الوصول ظاهرة في الكود الأمامي |
| **تخزين في localStorage** | 🔴 حرجة | بيانات حساسة مخزنة بشكل غير آمن |
| **غياب حماية CSRF** | 🟠 عالية | عدم وجود رموز حماية للنماذج |
| **خطر SQL Injection** | 🟠 عالية | استعلامات غير مؤمنة |
| **ثغرة XSS** | 🟠 عالية | استخدام `innerHTML` بدون تنقية |
| **غياب Rate Limiting** | 🟠 عالية | لا يوجد حد لمحاولات الدخول |

## 🛠️ التوصيات والإصلاحات المطلوبة

### إجراءات فورية (ضرورية قبل النشر)

1. **نقل المصادقة للخلفية**
   - إنشاء خادم خلفي (Node.js/Python/PHP)
   - نقل منطق التحقق من كلمات المرور
   - استخدام جلسات آمنة أو JWT

2. **حماية المفاتيح السرية**
   - إزالة جميع مفاتيح API من الكود الأمامي
   - استخدام متغيرات بيئة على الخادم
   - إنشاء Proxy API للوصول للخدمات الخارجية

3. **تأمين التخزين المحلي**
   - إزالة بيانات الاعتماد من localStorage
   - استخدام تخزين مشفر إذا لزم الأمر
   - تقليل البيانات المخزنة محلياً

4. **معالجة مدخلات المستخدم**
   - استبدال `innerHTML` بـ `textContent`
   - تطبيق تنقية صارمة للمدخلات
   - استخدام مكتبات مثل DOMPurify

## 🔧 التطوير

### إضافة وحدة جديدة

1. أنشئ ملف JavaScript جديد في مجلد `ecofine/`
2. عرف الوحدة باستخدام النمط المعياري:
   ```javascript
   const XModuleName = (function() {
       // كود الوحدة
       return {
           init: function() { },
           // دوال عامة أخرى
       };
   })();
   ```
3. ضمّن الملف في `index.html`
4. استدعِ `XModuleName.init()` في `app.js`

### اختبار النظام

```bash
# فحص الأخطاء في المتصفح
افتح أدوات المطور → Console

# اختبار المزامنة
1. افصل الاتصال بالإنترنت
2. نفذ عمليات في النظام
3. أعد الاتصال وتحقق من المزامنة
```
